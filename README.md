# MangaDex ReadmeList

GitHub Action for writing your MangaDex library statuses into a README, based on the marker-driven idea from [MAL-ReadmeList](https://github.com/gamersindo1223/MAL-ReadmeList).

It reads your real MangaDex library statuses such as `reading` and `plan_to_read`, renders matching README blocks, sorts by recent chapter activity by default, and can rotate your MangaDex refresh token back into GitHub Secrets.

See [docs.md](docs.md) for the full marker, input, and setup reference.

## README markers

Add one or more marker blocks to the markdown file you want to update:

```html
<!-- MANGADEX_READING:LIST -->
<!-- MANGADEX_READING:LIST_END -->

<!-- MANGADEX_PLAN_TO_READ:GRID -->
<!-- MANGADEX_PLAN_TO_READ:GRID_END -->
```

Statuses:

| Marker status | MangaDex status |
| --- | --- |
| `READING` | `reading` |
| `ON_HOLD` | `on_hold` |
| `PLAN_TO_READ` | `plan_to_read` |
| `DROPPED` | `dropped` |
| `RE_READING` | `re_reading` |
| `COMPLETED` | `completed` |

Display types:

| Type | Output |
| --- | --- |
| `LIST` | Markdown list with 50px covers |
| `GRID` | Linked cover grid |
| `DEFAULT` | Collapsible list |

## Installation

### Step 1. Add README markers

Add one or more marker blocks to the README or markdown file you want this action to update:

```html
<!-- MANGADEX_READING:LIST -->
<!-- MANGADEX_READING:LIST_END -->

<!-- MANGADEX_PLAN_TO_READ:GRID -->
<!-- MANGADEX_PLAN_TO_READ:GRID_END -->
```

### Step 2. Create a MangaDex personal client

1. Log in to MangaDex.
2. Open MangaDex settings.
3. Create a personal API client.
4. Save the client id and client secret.

### Step 3. Open the setup wizard

Open the plain HTML setup wizard.

Simple setup:

```text
http://setup.brtree.dpdns.org/web/
```

Local file:

```text
web/index.html
```

Fill in your MangaDex credentials, client details, target repository, branch, and markdown file. Click **Generate refresh token**, then copy the generated secret values and workflow snippet.

> [!NOTE]
> The setup wizard uses a CORS proxy based on [gamersindo1223/cors](https://github.com/gamersindo1223/cors) so plain HTML can call MangaDex auth from the browser. The proxy can see the MangaDex password, client secret, and refresh token during setup. Use a proxy instance you control or trust. The proxy must preserve `Content-Type: application/x-www-form-urlencoded`; [`proxy/cors-worker.ts`](proxy/cors-worker.ts) contains the fixed worker source for MangaDex OAuth.

If you do not want to use a browser proxy, generate the first refresh token with the local helper instead:

```powershell
node scripts/get-refresh-token.js
```

The helper prompts for MangaDex username, password, optional 2FA code, client id, and client secret. Password and client secret input are hidden in the terminal. 2FA is only needed for the initial login; scheduled workflow runs use `MD_REFRESH_TOKEN`, `MD_CLIENT_ID`, and `MD_CLIENT_SECRET`.

### Step 4. Add GitHub repository secrets

Open your target GitHub repository, then go to:

```text
Settings > Secrets and variables > Actions > New repository secret
```

Create these secrets:

| Secret | Purpose |
| --- | --- |
| `MD_CLIENT_ID` | MangaDex personal client id |
| `MD_CLIENT_SECRET` | MangaDex personal client secret |
| `MD_REFRESH_TOKEN` | Refresh token generated by the setup wizard or local helper |

To enable automatic refresh-token rotation, create a fine-grained GitHub PAT with repository `Secrets: Read and write` permission and save it as:

| Secret | Purpose |
| --- | --- |
| `GH_PAT` | Allows the workflow to update `MD_REFRESH_TOKEN` after MangaDex rotates it |

### Step 5. Create the GitHub Actions workflow

Create this file in your target repository:

```text
.github/workflows/mangadex-readlist.yml
```

Use the workflow generated by `web/index.html`, or start with this:

```yaml
name: Update MangaDex readlist

on:
  schedule:
    # Recommended interval: 24 hours. Please do not run more often to avoid abusing MangaDex.
    - cron: "0 0 * * *"
  workflow_dispatch:

jobs:
  update-readme:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - name: MangaDex ReadmeList
        uses: gamersindo1223/Mangadex-Readmelist@main
        with:
          md_client_id: ${{ secrets.MD_CLIENT_ID }}
          md_client_secret: ${{ secrets.MD_CLIENT_SECRET }}
          md_refresh_token: ${{ secrets.MD_REFRESH_TOKEN }}
          repo_path: "OWNER/REPO"
          repo_branch: "main"
          repo_filename: "README.md"
          display_limit: "10"
          sort_by: "latestUploadedChapter"
          sort_order: "desc"
          cover_url_format: "https://uploads.mangadex.org/covers/${mangaId}/${coverFileName}.256.jpg"
          cover_proxy_url: "https://apsiknb-image.hf.space${path}"
          rotate_secret: "true"
          gh_pat: ${{ secrets.GH_PAT }}
```

If you do not want automatic token rotation, set `rotate_secret: "false"` and omit `gh_pat`. When MangaDex eventually expires the old refresh token, rerun the local helper and update `MD_REFRESH_TOKEN`.

### Step 6. Run it

1. Commit and push your README markers and workflow file.
2. Open the repository's **Actions** tab.
3. Select **Update MangaDex readlist**.
4. Click **Run workflow**.

## Inputs

| Input | Default | Description |
| --- | --- | --- |
| `md_client_id` | required | MangaDex personal client id |
| `md_client_secret` | required | MangaDex personal client secret |
| `md_refresh_token` | required | MangaDex refresh token |
| `gh_token` | current workflow token | Token used for checkout, commit, and push |
| `gh_pat` | empty | PAT used only for secret rotation |
| `repo_path` | current repository | Repository to update |
| `repo_branch` | current ref | Branch to update |
| `repo_filename` | `README.md` | Markdown file to update |
| `display_limit` | `10` | Max entries per marker block, 1-100 |
| `sort_by` | `latestUploadedChapter` | One of `latestUploadedChapter`, `updatedAt`, `createdAt`, `title`, `followedCount`, `year`. |
| `sort_order` | `desc` | `asc` or `desc` |
| `title_language` | `en` | Preferred title language |
| `fallback_title_language` | `ja-ro` | Fallback title language |
| `content_ratings` | `safe,suggestive,erotica` | MangaDex content ratings to include |
| `cover_url_format` | `uploads.mangadex.org` template | Cover image URL template |
| `cover_proxy_url` | empty | Optional proxy template for cover image URLs |
| `metadata_format` | built-in | Optional metadata format for `LIST` and `DEFAULT` items |
| `metadata_format_reading` | unset | Optional metadata format override for `READING` |
| `metadata_format_plan_to_read` | unset | Optional metadata format override for `PLAN_TO_READ` |
| `empty_message` | built-in | Optional message for empty marker blocks |
| `empty_message_re_reading` | unset | Optional empty message override for `RE_READING` |
| `rotate_secret` | `false` | Update the refresh token repository secret |
| `refresh_token_secret_name` | `MD_REFRESH_TOKEN` | Secret to update when rotation is enabled |
| `commit_message` | `docs: update MangaDex readlist` | Commit message |

Metadata formats support placeholders such as `${status}`, `${lastChapter}`, `${lastRead}`, `${progress}`, `${year}`, and `${title}`. Unset per-status formats fall back to `metadata_format`; an explicitly empty value hides metadata for that status.

Cover URLs default to `https://uploads.mangadex.org/covers/${mangaId}/${coverFileName}.256.jpg`. The `https://mangadex.org/covers/...` route can return MangaDex's anti-hotlink placeholder image in browser markdown previews, so it is not the default.

If a markdown preview still hotlinks covers incorrectly, deploy [proxy/mangadex-cover-worker.js](proxy/mangadex-cover-worker.js) as a small Cloudflare Worker and set:

```yaml
cover_proxy_url: "https://apsiknb-image.hf.space${path}"
```

For a general CORS proxy, use:

```yaml
cover_proxy_url: "https://cor.brtree.dpdns.org/${url}"
```

Empty messages support `${status}` and `${libraryStatus}`. Unset per-status messages fall back to `empty_message`; an explicitly empty value hides the empty block text for that status.

## Development

```powershell
npm test
```

If MangaDex requests time out locally, run:

```powershell
npm run diagnose:network
```

With `MANGADEX_DOH=true` or `MANGADEX_DNS=1.1.1.1`, MangaDex requests use Cloudflare DNS-over-HTTPS through undici before connecting.

### Local `.env` testing

For local runs, copy `.env.example` to `.env` and fill in your MangaDex values:

```powershell
Copy-Item .env.example .env
node src/index.js
```

The action entrypoint loads `.env` automatically before reading inputs. Existing process environment variables win over `.env` values. You can use either `INPUT_MD_CLIENT_ID` / `INPUT_MD_CLIENT_SECRET` / `INPUT_MD_REFRESH_TOKEN` or `MD_CLIENT_ID` / `MD_CLIENT_SECRET` / `MD_REFRESH_TOKEN`. Set `MANGADEX_READLIST_ENV=false` to disable `.env` loading.

Use `test/fixtures/README.md` as a profile README fixture. It contains every supported MangaDex status with `LIST`, `GRID`, and `DEFAULT` marker blocks.

Render the fixture with your real MangaDex library from `.env`:

```powershell
npm run render:fixture
```

The generated preview is written to `test/fixtures/output/README.generated.md`.
If MangaDex rotates the refresh token, this command updates local `.env` automatically. Set `MANGADEX_UPDATE_ENV_REFRESH_TOKEN=false` to disable that.
